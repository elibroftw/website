The Copenhagen Book Git: Version Control System (Code History)

Here are some git commands I need to use but are hard to remember

  • diff3
  • To print only the current branch name
    git rev-parse --abbrev-ref HEAD
  • To reorder and squash commits
    • Ensure you do not have the default branch checked out
    • git rebase --interactive $UNTOUCHED_COMMIT
    • In the editor that opens up, use squash to merge a commit with its predecessor
    • git push --force
  • If you are on a non-collaborating branch, and want to group your changes (committed and working) differently,
    • First pick a commit you don't want to touch
    • git reset --soft $UNTOUCHED_COMMIT
      I prefer soft over mixed because I want to pick which changes are staged
    • Commit your changes as desired
    • git push --force
  • If you want to sync with a remote branch that had its history rewritten (e.g. coworker force pushed) and you have commits on local but not on the remote (i.e. git rebase --hard will not work). --hard gets rid of your local commits.
    git reset --hard @{u}
  • Shows reference logs of commits that were updated to the repository
    git reflog --before=2024-10-23T21:41:24.107351Z
  • Git Trailers (reading)
  • Make a shell file executable once it has been staged or committed
    git update-index --chmod=+x foo.sh
  • Updating the date of an commit amendment (commit --amend).
    • Linux
      git commit --amend --date "$(date -R)"
    • Windows & MacOS
      git commit --amend --date "$(date)"
  • See remote info, including the default branch
    git remote show origin

Full Stack / CRUD Developer

Database: PostgreSQL, use a managed PostgreSQL in production.

Backend: ASP.NET Core. Great performance, and comes with a lot of DX out of the box

When creating an ASP.NET project, use the dotnet CLI to select the WebAPI template for SPAs or the Web App template for Razor pages

If you use another backend framework, make the responses are as expected if the database restarts

You can install .NET either through Visual Studio C# Dev Kit, dotnet.microsoft.com, or Visual Studio

Frontend: use Vite to create a React or SolidJS TypeScript project combined with Inertia so that you are free to use any backend and still get server-side rendering for your single-page apps. Alternatively you can use Astro + API or the native SSR framework like Razor pages, Handlebars, htmx, Jinja2
